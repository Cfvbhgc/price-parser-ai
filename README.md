# AI Price Parser — умный разбор и категоризация прайс-листов

Инструмент превращает **кривой прайс-лист поставщика** (Excel/CSV произвольной
структуры — с мусорной шапкой, объединёнными ячейками и «плавающими» колонками)
в **аккуратный категоризированный Excel-отчёт**.

Поставщики присылают прайсы по-разному: у одного колонки `Наименование | Артикул
| Цена`, у другого `Код | Товар | Кол-во | Стоимость`, сверху — название фирмы,
телефоны и пустые строки. Жёстко прописать «цена в колонке C» нельзя — на
следующем файле всё поедет. Этот инструмент **находит структуру сам**.

Проект сделан «по-настоящему»: работает на приложенных примерах без заглушек.

---

## Что умеет

- 📥 **Вход:** Excel (`.xlsx`) и `.csv` произвольной структуры.
- 🧭 **Умный разбор без привязки к номерам колонок:**
  - динамически находит строку-заголовок по ключевым словам
    (Наименование/Название, Артикул/Код, Цена, Кол-во);
  - разворачивает объединённые ячейки;
  - отсекает мусорные строки сверху и строки-разделители внутри.
- 🤖 **Определение структуры через LLM (DeepSeek по умолчанию):** в модель уходит
  **только образец** (заголовок + несколько строк), а не весь файл — она отвечает
  компактным JSON-маппингом «где какая колонка». Дальше весь файл разбирается
  кодом. Экономно по токенам: **один вызов LLM на файл, а не на строку.**
  Без ключа API инструмент всё равно работает — структуру определяет
  эвристика по ключевым словам.
- 🏷 **Категоризация** по настраиваемому списку (крепёж, электрика, сантехника,
  инструмент — правится в `categories.json`).
- 📊 **Выход:** красивый Excel-отчёт (openpyxl) — колонки `Название · Артикул ·
  Цена · Категория`, тёмная шапка, зебра, сетка, автофильтр, закреплённая шапка.
- 🖥 **Консольный запуск:** путь к файлу + JSON с параметрами → готовый отчёт или
  понятное описание ошибки.

## Быстрый старт

Нужен **Python 3.11+**.

```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 1. Сгенерировать примеры кривых прайсов (кладутся в data/samples/)
python -m scripts.make_samples

# 2. Разобрать прайс и получить отчёт (по умолчанию → data/reports/)
python -m app.cli data/samples/postavshik2_sbornyj.xlsx
```

Готовые отчёты по примерам уже лежат в `data/reports/`.

### Ключ LLM (необязательно)

```bash
cp .env.example .env
# впишите LLM_API_KEY (ключ DeepSeek: https://platform.deepseek.com)
```

Без ключа определение структуры идёт по эвристике — на примерах результат тот же.
С ключом структуру определяет модель (полезно на нестандартных заголовках).
В выводе всегда видно, что сработало: `Структура определена через: llm | эвристика`.

## Запуск из консоли

```bash
python -m app.cli <путь_к_файлу> [JSON-параметры]
```

Примеры:

```bash
python -m app.cli price.xlsx
python -m app.cli price.xlsx '{"use_llm": false}'
python -m app.cli price.xlsx '{"output": "готово.xlsx", "sheet": "Прайс"}'
python -m app.cli postavshik.csv '{"report_title": "Прайс СтройСнаб, июль"}'
```

Параметры JSON:

| Параметр | Тип | Смысл | По умолчанию |
|----------|-----|-------|--------------|
| `output` | строка | Путь итогового отчёта | `data/reports/<имя>_отчёт.xlsx` |
| `use_llm` | bool | Определять структуру через LLM | `true` |
| `sheet` | строка/число | Лист Excel (имя или индекс) | активный лист |
| `categories_file` | строка | Свой файл категорий | `categories.json` |
| `report_title` | строка | Заголовок в отчёте | «Отчёт по прайс-листу» |

Вывод — путь к отчёту и статистика (сколько позиций, где найден заголовок, как
определена структура, разбивка по категориям). При ошибке — понятное сообщение и
код возврата `1`, без стектрейса.

## Как это устроено

Подробно и «на пальцах» — в [ARCHITECTURE.md](ARCHITECTURE.md). Кратко, путь файла:

```
   прайс (.xlsx/.csv)
        │  reader.py — читаем в «сетку», разворачиваем объединённые ячейки
        ▼
   header_detector.py — ищем строку-заголовок по ключевым словам, мусор сверху отсекаем
        │
        ▼
   structure.py ──► LLM (llm/schema_detector.py) даёт JSON-маппинг колонок
        │            └─ нет ключа / ошибка → эвристика (column_mapper.py)
        ▼
   extractor.py — по маппингу разбираем ВЕСЬ файл кодом, парсим цену, чистим мусор
        │
        ▼
   categorizer.py — категория по ключевым словам (categories.json)
        │
        ▼
   report.py — красивый Excel (тёмная шапка, зебра, сетка, автофильтр)
```

### Структура кода

| Модуль | Роль |
|--------|------|
| `app/reader.py` | Чтение xlsx/csv в сетку, разворот объединённых ячеек |
| `app/keywords.py` | Ключевые слова колонок (общие для поиска шапки и эвристики) |
| `app/header_detector.py` | Поиск строки-заголовка по скорингу ключевых слов |
| `app/llm/` | LLM-провайдер (DeepSeek/OpenAI) + определение маппинга по образцу |
| `app/column_mapper.py` | Эвристический маппинг колонок (фолбэк без LLM) |
| `app/structure.py` | Оркестрация: заголовок + маппинг (LLM → эвристика) |
| `app/extractor.py` | Разбор всего файла по маппингу + парсинг цены |
| `app/categorizer.py` | Категоризация по ключевым словам |
| `app/report.py` | Формирование стилизованного Excel-отчёта |
| `app/pipeline.py` | Полный конвейер + сбор статистики |
| `app/cli.py` | Консольный интерфейс |
| `scripts/make_samples.py` | Генерация примеров кривых прайсов |

## Настройка категорий

`categories.json` — список категорий с ключевыми словами:

```json
{
  "default_category": "Прочее",
  "categories": [
    {"name": "Крепёж", "keywords": ["болт", "гайка", "саморез", "дюбель"]},
    {"name": "Электрика", "keywords": ["кабель", "розетк", "автомат"]}
  ]
}
```

Слова сравниваются по вхождению в нижнем регистре (ловятся формы: «болт»,
«болты», «болтов»). При конфликте выигрывает **более длинное** ключевое слово —
поэтому «шуруповёрт» попадёт в «Инструмент», а не в «Крепёж» из-за слова «шуруп».
